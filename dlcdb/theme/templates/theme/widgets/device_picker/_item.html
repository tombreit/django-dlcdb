{% load i18n %} {% comment %} Shared device option card for the centralized device picker, driven by theme/js/picker.js. Reused for live-search results and the selected-device slot across every picker source (assets relocate, lending quick lend, …). The card is device-only: the option id is always the device pk and the meta line is always the device's active-record status badge — there is no per-flow branch. Generic device data attributes are emitted unconditionally; a consumer's glue JS reads only the ones it needs: data-room-id / data-room-label the device's current room (lending prefills the target room select from it). data-has-profile "1" when the device's type has a lending profile (only the lending source annotates this; absent -> falsy for other sources). data-lent "1" when the device is currently lent (relocate warns about moving a lent device). Params: device (core.Device) the device to render. The active record should be select_related for the status line / room. selectable (bool) search-result card (chevron) vs selected slot (button). locked (bool) selected slot is read-only (no change/remove button). multi_field_name (str) multi-select: emit a hidden input carrying the pk so the form submits the live set. {% endcomment %}
