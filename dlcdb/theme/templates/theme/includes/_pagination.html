{% comment %} Reusable numbered pager for filterbar list views. Caller passes: page_obj - a Page from theme.pagination.paginate() (carries .elided_page_range) Links use the native {% querystring %} tag, so every active search/filter/sort param is preserved and only `page` changes. Each link carries a real href (no-JS fallback) plus hx-get; hx-target / hx-swap / hx-push-url are inherited from the fragment root the pager is rendered inside (same trick as _sort_th.html), so a page click swaps the list fragment and pushes the URL. Beneath the pager a manual override toggles `?show_all=1`: unconditionally show every object, or return to the paginated view. It lives outside the num_pages guard because show-all collapses everything onto one page (num_pages == 1), which would otherwise hide the only way back. {% endcomment %} {% load i18n %}
