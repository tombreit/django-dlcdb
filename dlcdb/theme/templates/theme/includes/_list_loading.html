{% load i18n %} {% comment %} Loading veil for HTMX-swapped list fragments. Rendered as the first child of a fragment root that is `position-relative` and declares `hx-indicator=".htmx-indicator"`. HTMX's built-in `.htmx-indicator` / `.htmx-request` opacity CSS toggles it during any request that swaps the list (filter, search, sort, page, show_all). `pe-none` keeps it from intercepting clicks on the underlying rows (which use stretched-links). The spinner is `position-sticky` so it stays centered in the viewport band even when the request was triggered from the bottom pager of a long list. The inline `transition` delays the fade-in by 400ms so quick (sub-400ms) swaps never flash the veil: if the request finishes first, htmx removes `htmx-request` while opacity is still 0 and the fade never starts. Inline wins over htmx's own (delay-less, non-`!important`) indicator transition; htmx still drives opacity. {% endcomment %}
{# Centering lives on this wrapper: the spinner element must carry no `transform` utility (e.g. translate-middle-x), because Bootstrap sets those `!important`, and an !important transform overrides the spinner-border rotate animation — freezing the spinner. Flexbox centers it without touching transform. #}
